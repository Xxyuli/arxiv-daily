### **论文概要**

本文旨在解决机器人强化学习（RL）中因稀疏奖励信号导致的样本效率低下问题。作者提出了一种名为TOPReward的零样本、无需训练的过程奖励模型。该方法的核心创新在于，不要求预训练的视频-语言模型（VLM）直接生成数值化的任务进度值，而是通过分析模型内部对“任务是否完成”这一二元判断的**词元概率**，来提取一个连续、单调递增的进度信号。论文在包含130多个真实世界任务的ManiRewardBench基准测试和Open X-Embodiment数据集上验证了TOPReward的有效性，其性能显著优于现有零样本基线。此外，作者展示了TOPReward在成功检测和奖励对齐的行为克隆等下游应用中的实用性。

### **研究动机**

当前，视觉-语言-动作（VLA）模型在机器人领域的预训练方面进展迅速，但其在强化学习中的应用仍受限于真实环境中极低的样本效率和稀疏的奖励信号。为了提供细粒度的反馈，研究社区转向开发可泛化的过程奖励模型（第1节）。现有方法主要分为两类：1）在精心策划的机器人数据集上微调VLM作为奖励函数（如RoboReward、RoboDopamine），但这需要大量特定任务的数据收集，且泛化能力有限，难以跨越不同的机器人本体和视角（第1节，引用了Lee et al., 2026; Tan et al., 2025）；2）利用预训练VLM进行零样本奖励估计，如生成式价值学习（GVL）方法（Ma et al., 2024）。然而，GVL将进度估计视为视觉问答，要求模型为打乱的轨迹帧直接输出0-1的进度数值。这种方法仅在GPT-4、Gemini等闭源模型上表现良好，在开源VLM上则近乎失效（第1节，引用了Budzianowski et al., 2025）。

作者认为，开源VLM在进度估计上的失败，并非源于其缺乏对视频时序的理解能力，而是源于**文本生成的表示瓶颈**（第1节）。具体而言，VLM在遵循指令和表示数值词元方面存在不一致性和众所周知的偏差（第2节，引用了Wallace et al., 2019; Yuan et al., 2023）。因此，与其强迫VLM生成不稳定的数值输出，不如绕过文本生成，直接探测模型通过其内部表示（如词元概率）所蕴含的“世界知识”和“信念”。这一动机在自然语言处理领域已有先例，即模型内部激活比其生成的文本更能可靠地追踪其确定性和事实准确性（第2节，引用了Kadavath et al., 2022; Azaria & Mitchell, 2023）。本文旨在将这一原理系统性地应用于机器人奖励建模，以解锁开源VLM中已存在但未被利用的零样本奖励估计能力。

### **核心贡献与创新点**

本文的核心贡献与创新点可归纳为以下四个方面：

1.  **提出了一种基于词元概率的零样本进度估计新范式**：这是本文最核心的概念性创新。与现有方法（如GVL）要求VLM生成结构化数值输出不同，TOPReward将进度估计重新表述为一个**二元完任务判断的概率查询**（第3.1节）。具体而言，给定指令和视频轨迹前缀，模型被提示回答“上述视频是否完成了任务？答案是：True”。TOPReward不关心模型生成的文本答案，而是提取模型对肯定答案词元（如“True”）的**对数概率**作为奖励信号（公式(1)）。这种方法完全规避了VLM在数值生成和复杂指令遵循方面的弱点，直接利用其隐式的视频理解能力来评估任务完成的可能性。

2.  **设计并开源了ManiRewardBench基准测试**：为了对奖励模型进行严格、全面的评估，作者构建并引入了ManiRewardBench（第4节）。该基准包含超过130个独特的真实世界操作任务，覆盖四种不同的机器人平台（Franka, SO-100/101, 单臂/双臂YAM）。其关键创新在于提供了**细粒度的、阶段感知的时间标注**，即为每个任务片段手动标注了预定义子任务的开始和结束时间。这使得评估能够超越粗粒度的成功/失败判断，真正检验奖励模型对任务进展的敏感度。该基准为未来相关研究提供了重要的评估工具。

3.  **系统性地揭示了VOC指标的局限性并提出了解决方案**：论文通过理论分析和实验（第5.2节，图5）明确指出，广泛使用的价值顺序相关性（VOC）指标存在固有缺陷。VOC仅衡量预测值的排序相关性，而不关心绝对完成度。因此，一个早期达到平台期（例如仅完成30%）的失败轨迹，只要预测值单调递增，仍可获得高VOC分数，这使其无法有效区分成功与失败轨迹。TOPReward基于概率的方法天然提供了绝对完成度估计，从而在成功检测任务上显著优于依赖VOC的GVL方法（表3），为解决这一评估瓶颈提供了新思路。

4.  **验证了TOPReward作为通用奖励信号在下游任务中的有效性**：作者不仅将TOPReward作为评估工具，还将其集成到机器人学习流程中，展示了其实用价值（第5.3节）。通过将TOPReward估计的进度值转换为优势权重，并应用于优势加权回归（AWR）进行行为克隆微调，在六个真实世界的SO-100机器人任务上，该方法持续优于标准的无权重行为克隆基线（表4，图7）。这证明了TOPReward提取的奖励信号能够有效指导策略改进。

### **方法概述**

TOPReward方法的核心流程清晰，主要包含以下三个步骤：

**1. 基于词元概率的奖励计算（第3.1节）：**
给定一个任务指令 \(x\) 和一个视频轨迹前缀 \(\tau_{1:t} = (I_1, ..., I_t)\)，TOPReward的目标是计算一个标量奖励 \(r_t\)。首先，构建一个将判断基于视频的提示 \(u\)：
`“<|video|> The above video shows a robot manipulation trajectory that completes the following task: {INSTRUCTION}. Decide whether the above statement is True or not. The answer is: {a}”`
其中 `{a}` 被预设为词元序列 “True”。设预训练VLM的参数为 \(\theta\)，其定义了下一个词元的分布 \(p_\theta\)。奖励 \(r_t\) 定义为给定视频条件上下文 \(c(\tau_{1:t}, u)\) 时，模型对答案 “True” 的对数概率：
\[
r_t = \log p_\theta(a = \text{“True”} \mid c(\tau_{1:t}, u))
\]
作者发现布尔词元（True/False）在成功与失败片段间表现出最清晰的概率分离（第3.1节）。此步骤完全避免了自回归文本生成。

**2. 从轨迹前缀进行进度估计（第3.2节）：**

### **研究动机**

本文的研究动机源于在现实世界场景中对大型VLA模型进行高效、稳定强化学习（RL）后训练时所面临的核心矛盾。VLA模型结合了模仿学习与RL，已被证明能显著提升任务性能，其中价值函数作为评估动作质量、提供优化信号的核心组件至关重要（见第I节）。然而，现实世界VLA系统的RL训练面临数据稀缺、环境并行度有限、观测不完整等实际约束，使得大规模数据收集成本高昂，迭代缓慢。

更关键的是，现实世界VLA数据集通常由异构数据源混合而成，包括在线部署的策略采样、人类操作员针对错误或不安全行为的纠正性干预，以及来自早期策略或预收集演示的历史数据（见第I节）。在这种设置下，对当前VLA策略的价值估计本质上是一个**离策略问题**。但直接应用传统的离策略TD学习会因分布偏移和误差累积而导致估计偏差和学习不稳定（见第I节，引用[19, 47]）。

为了维持高容量VLA系统的训练稳定性，先前工作（如AWR [36]、RECAP [14]）通常采用**保守的、基于同策略价值估计的学习策略**（见第I节）。这些方法中的价值函数并非直接评估被优化的当前策略，而是估计由混合数据诱导出的一个隐含策略的期望回报。这种设计依赖于一个**强假设**：演化中的策略的行为分布与回放缓冲区中代表的混合分布保持接近（见第I节）。在现实世界VLA设置中，人类干预、策略更新和环境变化会持续改变数据分布，此假设尤为脆弱。当策略行为显著偏离历史数据分布时（例如，学习到新的恢复行为），基于历史数据训练的价值函数可能无法反映当前策略的真实价值，从而提供误导性的优化信号，抑制对鲁棒操作至关重要的探索（见第I节）。

这种妥协导致了现实世界VLA训练中**鲁棒性与学习效率之间的根本权衡**。因此，本文旨在探究并解决一个核心问题：**何时以及如何能够可靠地将离策略强化学习应用于改进现实世界的VLA策略？**（见第I节末尾）。作者认为，先前方法的保守性并非现实世界VLA训练所固有，而是源于对价值估计获取和使用方式的特定假设。通过明确解决价值函数偏差的关键来源，可以在不损害训练稳定性的前提下部分放宽这些假设，从而重新引入更高效的离策略学习。

### **核心贡献与创新点**

论文的研究动机源于将通用VLA模型可靠部署到真实世界所面临的三个核心障碍（见第1节）。首先，**速度不足**：现有VLA系统通常产生保守、时间密集的动作，未能充分利用快速自由空间运动，闭环反应性有限，导致吞吐量低下（引用[28, 18, 2]）。其次，**评估协议不匹配**：主流评估仍侧重于在精心重置条件下的单次尝试成功率（引用[31]）。这种评估掩盖了在长时间连续运行中，由于任务重复、初始状态漂移和小误差累积所导致的性能退化，无法衡量系统在需要外部干预前能持续运行多久。第三，**端到端延迟**：大型VLA堆栈通常需要大量计算或依赖网络组件，增加了延迟，在需要快速反应时降低了可预测性（引用[51, 6, 26]）。

作者进一步指出，单次尝试成功率作为一个单一的评估指标存在根本性缺陷（第1节）。它将速度-精度权衡和不同类型的失败模式压缩为一个数字，并且对长期性能退化和干预频率不敏感。在实际部署中，操作上更关心的是系统能否在没有外部帮助的情况下持续运行（引用[7]）。因此，需要一个能够同时反映**生产力**和**可靠性**的评估框架，以捕捉主导真实部署的权衡关系。这些部署需求（快速运动、长时运行、设备端运行）和评估指标的缺失，共同构成了本研究的核心动机。

#### **3. 核心贡献与创新点**
本文的核心贡献与创新点主要体现在评估框架、方法设计和系统集成三个层面：

1.  **生产力-可靠性平面评估框架**：论文提出了一个面向部署的二维评估平面，包含**任务每小时完成数**和**平均无干预时间**两个指标，并定义了**连续运行协议**（见第2节）。TPH衡量在固定时间窗口内成功完成的任务数量，直接反映“快速运动”能力；MTBI衡量两次需要外部干预事件之间的平均运行时间，直接反映“长时运行”能力。该框架将评估重点从独立的单次尝试转移到连续的、状态会漂移的长时间运行，更贴近实际生产场景。

2.  **面向连续运行的数据策略**：
    *   **游戏数据预训练**：大规模收集无结构、无任务标签的游戏数据，用于**语言无条件预训练**，以学习任务无关的、鲁棒的物理交互先验，为模型提供恢复行为的基础（见第3.3.2节）。
    *   **循环任务数据集**：不同于标准的“成功即停止”的非循环数据，论文收集**连续、无重置的重复任务演示轨迹**（见第3.3.3节）。这种数据形式让模型学习处理周期之间的状态转换和累积漂移，是获得“长时运行”能力的关键。
    *   **空间感知下采样**：应用ESPADA方法（引用[28]），利用VLM-LLM管道根据3D夹爪-物体空间关系将轨迹语义分割为“随意”和“精确”阶段，并对“随意”阶段进行**选择性加速压缩**，同时保持“精确”阶段的高频控制，从而提升运动速度（见第3.3.4节及公式(9)）。

3.  **高效且可调的设备端推理系统**：
    *   **整流流蒸馏**：采用整流流技术，将一个多步（≥10步）的流匹配教师模型**蒸馏为一个仅需两步推理的学生模型**（见第3.2.2节及公式(6)）。这大幅降低了推理延迟，为在设备端实现高频控制腾出了计算预算。
    *   **分类器无关引导作为部署旋钮**：在训练中通过随机掩码指令学习无条件预测，在推理时通过线性插值条件预测和无条件预测，实现CFG（见第3.2.3节及公式(7)）。引导强度`w`作为一个可调参数，允许在部署时动态平衡**指令遵循**和**学习到的交互先验**，从而在速度与可靠性之间进行权衡。
    *   **高频控制**：将蒸馏节省的延迟预算重新投资于**缩短动作块执行时域并增加重新推理的频率**（见第3.2.4节）。这种设计提高了闭环反应性，能更好地对抗漂移和时序误差的累积，从而提升MTBI。

Habilis-β采用前缀-后缀架构（见图2）。前缀是一个预训练的VLM，用于处理多视角图像和语言指令，生成融合的多模态表征。后缀是一个动作专家模型，以前缀的输出和本体感知状态为条件，生成未来H步的连续动作块。

**推理流程**（见图1右侧）：
1.  观测和指令经过VLM前缀处理。
2.  动作专家（蒸馏后的两步模型）以前缀输出和当前状态为条件，生成动作块。在此过程中，可**可选地应用CFG**：分别计算条件预测`v_cond`和无条件预测`v_uncond`，然后按`v_cfg = v_uncond + w * (v_cond - v_uncond)`进行混合，并进行**范数重缩放**（公式(8)）以稳定训练。
3.  执行缩短后的动作块（得益于ESPADA和高频控制设计）。
4.  在更短的时间间隔后，重新进行感知-推理-执行循环，实现高频闭环控制。

整个感知-动作循环在设备端运行，以消除网络延迟。

            arXiv:2602.19506v1 Announce Type: cross  Abstract: Feature caching approaches accelerate diffusion transformers (DiTs) by storing the output features of computationally expensive modules at certain timesteps, and exploiting them for subsequent steps to reduce redundant computations. Recent forecasting-based caching approaches employ temporal extrapolation techniques to approximate the output features with cached ones. Although effective, relying exclusively on temporal extrapolation still suffers from significant prediction errors, leading to performance degradation. Through a detailed analysis, we find that 1) these errors stem from the irregular magnitude of changes in the output features, and 2) an input feature of a module is strongly correlated with the corresponding output. Specifically, we introduce relational feature estimation (RFE) to estimate the magnitude of changes in the output features from the inputs, enabling more accurate feature predictions. We also present relational cache scheduling (RCS), which estimates the prediction errors using the input features and performs full computations only when the errors are expected to be substantial. ### **论文概要**
本文针对扩散变换器（DiT）推理加速中的特征缓存问题，提出了一种名为关系特征缓存（RFC）的新框架。现有基于时序外推的缓存方法因特征变化幅度的不规则性而产生显著预测误差。本文通过分析发现，模块的输入与输出特征变化高度相关。基于此，RFC包含两个核心组件：关系特征估计（RFE）利用输入特征变化来更准确地预测输出特征；关系缓存调度（RCS）则通过输入特征的预测误差动态决定何时执行完整计算。实验表明，RFC在多种DiT模型和生成任务上均显著优于现有方法，在保持图像/视频生成质量的同时，有效降低了计算开销。

### **研究动机**
扩散变换器（DiTs）在生成任务上表现出色，但其迭代去噪过程需要大量前向计算，导致推理速度慢、计算成本高。特征缓存方法通过缓存某些时间步的昂贵模块（如注意力、MLP）的输出特征，并在后续时间步复用，以减少冗余计算，是一种有前景的加速方案。

然而，现有方法存在明显不足。早期方法（如FORA）直接复用缓存特征，导致缓存误差随时间步累积，生成质量下降（见第1节）。近期基于预测的方法（如FasterCache, TaylorSeer）采用时序外推技术，假设特征随时间平滑演变。但本文通过详细分析（见图1(a)）发现，**输出特征的变化幅度在不同时间步之间存在显著的不规则性**，这使得单纯依赖时序外推仍会产生较大的预测误差（见图1(c)）。例如，即使是最先进的TaylorSeer，其预测误差相比直接复用的FORA仅略有降低，在完整计算间隔较大（即FLOPs较低）时，生成质量仍会严重下降（见图1(d)）。

作者进一步观察到，**模块输入特征的变化幅度与其输出特征的变化幅度高度相关**（对比图1(a)与图1(b)）。这一关键发现表明，输入特征可以作为预测输出特征变化的有效线索。因此，本文的研究动机是：**克服现有缓存方法因忽略特征变化不规则性而导致的预测误差问题，通过利用输入-输出特征之间的关系，设计一个更准确、更鲁棒的特征缓存框架**。

### **核心贡献与创新点**
本文的核心贡献是提出了关系特征缓存（RFC）框架，其创新点具体体现在两个新颖的组件上：

1.  **关系特征估计（RFE）：一种利用输入-输出关系来增强特征预测准确性的新方法。**
    *   **创新机制**：现有预测方法（如线性外推、泰勒展开）主要关注特征变化的“方向”，并假设其幅度遵循简单规律（如线性）。RFE的核心创新在于**显式地估计特征变化的“幅度”**。它定义了一个比率 `s_k(t-k)`，表示在时间步 `t-k` 处，输出特征变化与输入特征变化的L2范数之比（见公式(5)）。
    *   **理论依据与发现**：通过经验分析（图2(a)）和一个基于局部线性假设的命题（命题1，见第3.2节），作者证明了 `s_k(t-k)` 在不同时间步 `k` 上具有高度一致性（相对标准差约2%）。这意味着，可以利用最近一次完整计算步骤中计算出的比率 `s_N(t)`，结合当前时间步的输入特征变化 `∥∆_k I(t-k)∥_2`，来准确估计当前输出特征的变化幅度 `∥∆_k O(t-k)∥_2`（见公式(15)）。
    *   **与现有工作的区别**：与TaylorSeer（仅使用泰勒展开估计方向）或线性外推方法（使用固定或简单变化的缩放因子 `w(t)`，见第4.3节表5）不同，RFE**动态地、自适应地**根据输入特征的实际变化来调整对输出特征变化的幅度估计，从而更好地捕捉特征演化的不规则动态。

2.  **关系缓存调度（RCS）：一种基于输入特征预测误差动态触发完整计算的新策略。**
    *   **创新机制**：传统缓存方法使用固定的完整计算间隔（`N`），这无法适应不同时间步预测误差的波动。RCS的创新在于**将调度决策与预测误差的估计直接挂钩**。其核心思想是：输出特征的预测误差 `E_O` 与输入特征的预测误差 `E_I` 高度相关（见图2(b)）。
    *   **高效代理**：由于直接计算 `E_O` 需要昂贵的完整计算，RCS巧妙地**使用第一个模块的输入特征预测误差的相对L1范数 `Ē_I` 作为 `E_O` 的代理**（见公式(21)）。在采样过程中，RCS累积这些输入误差（见公式(22)），当累积值超过预设阈值 `τ` 时，则执行完整计算以重置误差。
    *   **与现有工作的区别**：不同于CacheMe等基于预定义误差的静态调度，或TeaCache等基于输入特征距离的触发机制，RCS是**首个将调度决策建立在预测误差（通过输入误差代理）累积基础上的动态方法**。表6的实验表明，基于误差的调度优于基于简单距离的调度。

**总结**：RFC的概念性创新在于**系统性地挖掘并利用了DiT中模块输入与输出特征变化之间的强相关性**，并据此设计了两个互补的组件（RFE用于更准的预测，RCS用于更智能的调度），共同构成了一个比单纯依赖时序外推更强大的缓存加速框架。

### **方法概述**
RFC框架在DiT的采样过程中运作，旨在减少注意力、MLP等昂贵模块的计算。其整体流程如下：

**1. 基础与观察**：
方法基于TaylorSeer的泰勒展开预测框架（公式(3)）。关键观察是：输入特征变化 `∥∆_k I∥_2` 与输出特征变化 `∥∆_k O∥_2` 的比率 `s_k` 在时间上基本恒定（图2(a)，命题1）。

**2. 关系特征估计（RFE）的运作流程**：
对于当前时间步 `t-k`（`k` 为距上次完整计算步 `t` 的偏移）：
*   **输入特征获取**：轻量计算当前模块的输入特征 `I(t-k)`（仅涉及LayerNorm、缩放等操作）。
*   **输入变化计算**：计算 `∥∆_k I(t-k)∥_2 = ∥I(t-k) - I(t)∥_2`。
*   **比率复用**：使用最近一次完整计算步（间隔为 `N`）计算出的固定比率 `s_N(t) = ∥∆_N O(t)∥_2 / ∥∆_N I(t)∥_2`。
*   **输出变化幅度估计**：根据公式(15)，估计 `∥∆_k O(t-k)∥_2 ≈ s_N(t) * ∥∆_k I(t-k)∥_2`。
*   **特征预测修正**：RFE不改变TaylorSeer预测的特征变化方向（即泰勒展开项的和），而是用上述估计的幅度对方向向量进行重新缩放。具体地，最终的预测输出特征 `O_RFE(t-k)` 由公式(16)给出：
    `O_RFE(t-k) = O(t) + [ s_N(t) * ∥∆_k I(t-k)∥_2 ] * g( Σ (k^i / i!) * (∆_N^i O(t) / N^i) )`
    其中 `g(·)` 是L2归一化函数（公式(17)），用于确保括号内的泰勒展开和项是单位方向向量。**RFE的创新在于用 `s_N(t) * ∥∆_k I(t-k)∥_2` 这个动态估计的标量，替换了原始泰勒展开中隐含的、基于固定间隔 `N` 和 `k` 的静态幅度估计**。

**3. 关系缓存调度（RCS）的运作流程**：

论文的研究动机源于对当前VLA模型在结构化、长视野操作任务中适用性的质疑，并特别关注其巨大的计算与能源成本（见第I节）。尽管VLA模型因其通过大规模预训练实现通用策略的潜力而受到关注，但作者指出，现有研究存在两个关键缺口。

**A. 神经符号模型（NSM）架构与流程（第III-D节）**
该模型基于Lorang等人的工作[2]，核心是分层架构：高层符号规划 + 底层神经控制。
1.  **符号抽象学习**：从少量原始演示轨迹 *D* 中，系统提取节点转移 *τ_node = (n, l, n‘)*，构建图 *G*。通过计算最小互模拟 *Ḡ* 来压缩状态空间。随后，使用基于Answer Set Programming的求解器（引用[21], [34]）从 *Ḡ* 中自动推断出一个PDDL形式的符号域 *σ = ⟨E, F, S, O⟩*。**这是关键创新**：NSM无需人工指定任务规则，而是从基础堆叠演示中自动学习汉诺塔的完整符号规划域。
2.  **物体检测**：为与VLA输入对齐，NSM同样只接收图像和本体感知数据。它使用YOLOv8检测器结合梯度提升回归器，从静态摄像头和腕部摄像头估计物体的3D位姿，进而计算末端执行器与物体的相对位姿，作为策略输入。
3.  **控制策略训练**：每个符号操作符 *o_i* 关联一个神经策略 *π_i*。每个技能进一步分解为动作步子策略 *π_i,j*（受选项框架启发[36]）。使用一个任务相关特征选择器 *φ* 过滤观测，只保留与当前操作符相关的物体，并将其坐标转换到末端执行器坐标系，以提高样本效率。底层策略采用扩散模型进行训练（引用[30]），以捕捉动作的多模态分布。
4.  **执行流程**：测试时，用户指定初始和目标状态，转换为PDDL任务 *T*。经典规划器（MetricFF）生成抽象计划 *P = [o_1, ..., o_|P|]*。然后依次调用每个 *o_i* 对应的策略 *π_i*，该策略内部顺序调用动作步子控制器 *π_i,j* 直至满足终止条件。
